# bg

> 将作业置于后台运行，使其在 shell 中继续执行但不占用终端。
> 更多信息：<https://manned.org/bg>

- 将最近挂起的作业置于后台：
  `bg`

- 将指定作业号的作业置于后台：
  `bg %{{作业号}}`

- 查看所有作业状态：
  `jobs`

- 挂起当前前台作业并置于后台：
  `Ctrl+Z` 然后 `bg`

- 启动命令并立即置于后台：
  `{{命令}} &`

- 启动命令后挂起并置于后台：
  `{{命令}}`
  `Ctrl+Z`
  `bg`

- 将多个作业置于后台：
  `bg %{{作业号1}} %{{作业号2}}`

- 查看后台作业的详细信息：
  `bg %{{作业号}} && jobs -l`

- 启动后台作业并记录作业号：
  `{{命令}} &`
  `jobs`

- 将所有挂起的作业置于后台：
  `for job in $(jobs -sp); do bg $job; done`

- 启动长时间运行任务并置于后台：
  `{{长时间任务}} &`
  `bg %+`

- 查看后台作业的进程 ID：
  `jobs -l`

- 将特定作业置于后台并发送信号：
  `bg %{{作业号}} && kill -USR1 %{{作业号}}`

- 启动批处理作业并管理：
  `{{批处理命令}} &`
  `jobs`
  `bg %{{作业号}}`

- 监控后台作业状态：
  `watch "jobs -l"`

- 将后台作业重新置于后台（如果已在前台）：
  `bg %+`

- 启动服务并置于后台：
  `{{服务启动命令}} &`
  `bg %+`

- 查看后台作业的资源使用：
  `bg %{{作业号}} && ps -p $(jobs -p %{{作业号}}) -o pid,pcpu,pmem,cmd`

- 将后台作业组置于后台：
  `bg %{{作业组号}}`

- 启动多个后台作业：
  `for i in {1..5}; do {{命令}}_$i &; done`
  `jobs`

- 将最近的后台作业重新置于后台：
  `bg %-`

- 启动后台作业并设置环境变量：
  `VAR={{值}} {{命令}} &`
  `bg %+`

- 查看后台作业的运行状态：
  `bg %{{作业号}} && jobs -s`

- 将后台作业从挂起状态恢复运行：
  `bg %{{作业号}}`

- 启动后台作业并重定向输出：
  `{{命令}} > {{输出文件}} 2>&1 &`
  `bg %+`

- 查看后台作业的完成状态：
  `wait %{{作业号}} && echo "作业完成"`

- 将后台作业重新调度：
  `bg %{{作业号}} && renice {{优先级}} $(jobs -p %{{作业号}})`

- 启动后台作业并发送通知：
  `{{命令}} &`
  `bg %+`
  `wait %{{作业号}} && notify-send "作业完成"`

- 批量管理后台作业：
  `jobs -p | xargs -I {} bg %{}`
