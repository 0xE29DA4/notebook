# nohup

> 让命令在退出终端后继续运行，忽略挂起信号。
> 更多信息：<https://manned.org/nohup>

- 在后台运行命令并忽略挂起信号：
  `nohup {{命令}} &`

- 运行命令并将输出重定向到指定文件：
  `nohup {{命令}} > {{输出文件}} &`

- 运行命令并将标准错误重定向到指定文件：
  `nohup {{命令}} 2> {{错误文件}} &`

- 将标准输出和错误都重定向到同一个文件：
  `nohup {{命令}} > {{输出文件}} 2>&1 &`

- 运行脚本并忽略所有输出：
  `nohup {{命令}} > /dev/null 2>&1 &`

- 运行长时间运行的数据处理任务：
  `nohup python {{数据处理脚本}}.py > {{日志文件}} 2>&1 &`

- 运行 Web 服务器：
  `nohup python -m http.server {{端口}} > {{服务器日志}} 2>&1 &`

- 运行数据库备份：
  `nohup mysqldump -u {{用户}} -p{{密码}} {{数据库}} > {{备份文件}} 2>&1 &`

- 运行文件同步任务：
  `nohup rsync -av {{源目录}} {{目标目录}} > {{同步日志}} 2>&1 &`

- 运行编译任务：
  `nohup make -j{{核心数}} > {{编译日志}} 2>&1 &`

- 运行机器学习训练：
  `nohup python {{训练脚本}}.py > {{训练日志}} 2>&1 &`

- 运行下载任务：
  `nohup wget -c {{URL}} > {{下载日志}} 2>&1 &`

- 运行批处理脚本：
  `nohup bash {{批处理脚本}}.sh > {{批处理日志}} 2>&1 &`

- 运行容器服务：
  `nohup docker run -d {{镜像名}} > {{容器日志}} 2>&1 &`

- 运行监控脚本：
  `nohup {{监控脚本}} > {{监控日志}} 2>&1 &`

- 使用 tee 同时输出到文件和屏幕：
  `nohup {{命令}} 2>&1 | tee {{日志文件}} &`

- 运行多个命令：
  `nohup bash -c "{{命令1}} && {{命令2}}" > {{日志文件}} 2>&1 &`

- 设置环境变量运行：
  `nohup env {{变量}}={{值}} {{命令}} > {{日志文件}} 2>&1 &`

- 使用特定用户运行：
  `nohup sudo -u {{用户名}} {{命令}} > {{日志文件}} 2>&1 &`

- 在特定目录下运行：
  `nohup {{命令}} > {{日志文件}} 2>&1 &`

- 查看正在运行的 nohup 任务：
  `ps aux | grep nohup`

- 终止 nohup 任务：
  `kill $(pgrep -f "{{命令特征}}")`

- 查看任务的实时输出：
  `tail -f {{日志文件}}`

- 查看任务的错误输出：
  `tail -f {{错误文件}}`

- 创建带时间戳的日志文件：
  `nohup {{命令}} > {{日志文件}}_$(date +%Y%m%d_%H%M%S) 2>&1 &`

- 运行带超时的命令：
  `nohup timeout {{秒数}} {{命令}} > {{日志文件}} 2>&1 &`

- 使用 screen 和 nohup 组合：
  `nohup screen -dmS {{会话名}} {{命令}}`

- 监控 nohup 任务的资源使用：
  `nohup {{命令}} > {{日志文件}} 2>&1 &`
  `ps aux | grep "{{命令特征}}" | awk '{print $2}' | xargs -I {} ps -p {} -o pid,pcpu,pmem,cmd`
