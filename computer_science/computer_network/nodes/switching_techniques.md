# 交換技術

- 电路交换
  - 用于电话网络
  - 从主叫端到被叫端建立一条专用的物理通路，在通信时间内，两用户始终占用端到端的线路资源
  - 优点
    - 传输速率高、实时性强
    - 数据无失序问题
  - 缺点
    - 通信前需要建立好电路连接
    - 建立/释放连接需要时间开销
    - 通信线路被独占，利用率低
    - 线路分配灵活性差
    - 不支持差错控制
  - 适用于低频次、大量的数据传输
- 报文交换
  - 用于电报网络
  - 优点
    - 通信前无需建立连接
    - 数据以报文为单位被交换节点间存储转发，通信线路可以灵活分配
    - 通信时间内，两个用户无需独占一整条物理线路，线路利用率高
    - 支持差错控制
  - 缺点
    - 报文不定长，不方便存储转发管理
    - 长报文的存储转发、缓存、差错控制开销大
    - 长报文出错重传代价高
- 分组交换
  - 用于现代计算机网络
  - 优点
    - 通信前无需建立连接
    - 数据以分组为单位被交换节点间存储转发，通信线路可以灵活分配
    - 通信时间内，两个用户无需独占一整条物理线路，线路利用率高
    - 支持差错控制
    - 分组定长，方便存储转发管理
    - 分组的存储转发、缓存、差错控制开销小
    - 分组出错重传代价低
  - 缺点
    - 增加了控制信息在数据中的占比
    - 报文拆分后，需要处理分组失序、丢失、重复等问题，增加了处理复杂度
- 虚电路交换
  - 本质上还是分组交换，但是要求通信前通信双方事先建立逻辑连接（虚电路），确保分组按序传输，并在通信结束后拆除连接
  - 把分组交换产生的失序、丢包等问题交给了网络核心部分而非边缘部分处理
