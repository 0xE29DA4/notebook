# fg

> 将后台作业置于前台运行，使其占用终端并可以交互。
> 更多信息：<https://manned.org/fg>

- 将最近的后台作业置于前台：
  `fg`

- 将指定作业号的作业置于前台：
  `fg %{{作业号}}`

- 查看所有作业状态：
  `jobs`

- 将最近的后台作业置于前台并监控：
  `fg %+`

- 将指定后台作业置于前台：
  `fg %{{作业号}}`

- 将最近的前台作业重新置于前台：
  `fg %-`

- 启动命令后置于后台，然后置于前台：
  `{{命令}} &`
  `fg %1`

- 将作业置于前台并查看详细信息：
  `fg %{{作业号}} && jobs -l`

- 将作业置于前台并发送信号：
  `fg %{{作业号}}` 然后 `Ctrl+C`

- 将作业置于前台并挂起：
  `fg %{{作业号}}` 然后 `Ctrl+Z`

- 查看前台作业的进程 ID：
  `fg %{{作业号}} && jobs -l`

- 将后台作业组置于前台：
  `fg %{{作业组号}}`

- 监控前台作业状态：
  `fg %{{作业号}}` 然后 `Ctrl+Z`
  `bg %{{作业号}}`

- 将作业置于前台并重新调度：
  `fg %{{作业号}}`
  `Ctrl+Z`
  `bg %{{作业号}} && renice {{优先级}} $(jobs -p %{{作业号}})`

- 将作业置于前台并等待完成：
  `fg %{{作业号}}`
  `wait %{{作业号}}`

- 将作业置于前台并记录状态：
  `fg %{{作业号}} && echo "作业 %{{作业号}} 已置于前台"`

- 将作业置于前台并设置环境变量：
  `VAR={{值}} fg %{{作业号}}`

- 将作业置于前台并查看资源使用：
  `fg %{{作业号}} &`
  `ps -p $! -o pid,pcpu,pmem,cmd`

- 将作业置于前台并记录日志：
  `fg %{{作业号}} 2>&1 | tee {{日志文件}}`

- 将作业置于前台并发送完成通知：
  `fg %{{作业号}}`
  `wait %{{作业号}} && notify-send "作业完成"`

- 批量管理前台作业：
  `for job in %1 %2 %3; do fg $job; done`

- 将作业置于前台并检查退出状态：
  `fg %{{作业号}}`
  `echo "退出状态: $?"`

- 将作业置于前台并限制时间：
  `timeout {{秒数}} fg %{{作业号}}`

- 将作业置于前台并忽略信号：
  `trap '' SIGINT SIGTERM`
  `fg %{{作业号}}`

- 将作业置于前台并捕获输出：
  `output=$(fg %{{作业号}})`

- 将作业置于前台并处理错误：
  `fg %{{作业号}} 2> {{错误文件}}`

- 将作业置于前台并记录时间：
  `time fg %{{作业号}}`

- 将作业置于前台并发送邮件通知：
  `fg %{{作业号}}`
  `echo "作业完成" | mail -s "任务通知" {{邮箱}}`

- 将作业置于前台并更新状态文件：
  `fg %{{作业号}}`
  `echo "$(date): 作业完成" >> {{状态文件}}`
